# OpenClaw

## Why It Is Experimental

OpenClaw is intentionally marked experimental because its platform model is broader than a repo-scoped coding CLI.

It is promising, but it pulls in:

- gateway concerns
- assistant-platform concerns
- skill and provider ecosystems beyond coding alone

## Why It Still Matters

OpenClaw is valuable research input for:

- gateway-style orchestration
- YAML or workspace-driven coordination
- skills and provider abstraction
- event-driven extension patterns

## Role In `agent-worktree`

The project should keep OpenClaw visible as a secondary adapter target, not as the first model that defines every public contract.
