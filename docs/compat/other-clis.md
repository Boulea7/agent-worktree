# Other CLIs

## Goal

The project should remain open to additional coding-agent CLIs, not just the named Tier 1 tools.

## Adapter Strategy

Future implementation should allow:

- native adapters for tools with rich, stable docs
- generic shell adapters for broader CLI compatibility

## Promotion Questions

A CLI should be evaluated against:

- project-level guidance support
- machine-readable output
- headless automation
- session resume or equivalent
- MCP, plugins, or extension surfaces
- safe repo-level configuration
